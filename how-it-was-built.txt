HOW THE NYC ECONOMIC MOBILITY DASHBOARD WAS BUILT
===================================================

CONTEXT
-------
This dashboard was built for the Office of the Deputy Mayor for Economic
Justice as an internal tool to track education, workforce, and economic
development outcomes for New York City. It is a static data dashboard
(no backend, no authentication) designed to be updated monthly/quarterly
with new data. Includes an AI chat widget for data Q&A.

The build was done entirely by Claude Code (claude.ai/code) across
multiple sessions, working in a local directory on macOS.

Powered by SquareOne (https://www.squareoneed.com/).
Deployed at: https://mobility.squareoneed.com/


CONSTRAINTS ENCOUNTERED
-----------------------
1. DISK SPACE: The machine had only 138MB free (99% full). A standard
   Vite + React + Recharts + Tailwind project requires ~400MB of
   node_modules. This ruled out npm-based tooling entirely.

2. NODE.JS: The installed Node.js binary was x86_64 on an arm64 Mac,
   so it wouldn't execute. The only working Node binary was bundled
   inside Cursor.app, used solely for syntax validation.

3. RECHARTS CDN FAILURE: The initial approach used React + Recharts +
   Babel loaded from CDN. Recharts' UMD build returned undefined.

4. NESTED TEMPLATE LITERALS: After switching to Chart.js, nested
   backtick strings caused syntax errors. The entire codebase was
   rewritten using pure string concatenation.

5. CHART.JS BAR CHART BUG: Overwriting Chart.defaults.scale.grid with
   a new object wiped internal properties needed for bar rendering.
   Fixed by setting individual properties instead.

6. API KEY SECURITY: GitHub push protection blocked commits containing
   the Anthropic API key. Solved by moving the key to config.js
   (gitignored) and loading via window.ANTHROPIC_KEY.


SOLUTION: SINGLE-FILE ARCHITECTURE
-----------------------------------
The dashboard is a single index.html file (~1280 lines) loading CDN
resources:

  - Tailwind CSS (cdn.tailwindcss.com) -- utility-first styling
  - Chart.js 4.4.4 (cdn.jsdelivr.net) -- all charts (line, bar, doughnut)
  - Inter font (Google Fonts) -- typography
  - Anthropic API (api.anthropic.com) -- AI chat responses

No React. No JSX. No Babel. No build step. No npm. Just vanilla
JavaScript with string-concatenation-based DOM rendering.

DASHBOARD STRUCTURE
-------------------
The dashboard has a landing page, four layers, a data & sources page,
and a floating AI chat widget. Navigation lives inside full-width hero
image bands on each page (not a sticky header).

LANDING PAGE (default view)
  - Hero: Brooklyn Bridge image, 420px, white nav + title
  - Framing paragraph about economic mobility
  - Four clickable layer cards with images and dynamic headline findings

LAYER 01 - PEOPLE & PIPELINE
  - Hero: Classroom image, 300px, guiding questions
  - "So What" narrative panel (auto-generated from data)
  - Pathway tree diagram (HTML/CSS, 4 stages from 9th grade to workforce)
  - 10 metric cards with sparklines, targets, equity disaggregation
  - Education trends line chart with 2019 baselines

LAYER 02 - LABOR MARKET & SECTORS
  - Hero: NYC buildings image, 300px, guiding questions
  - "So What" narrative panel
  - 6 metric cards with sparklines
  - Horizontal bar chart: sector employment change with wages
  - Bar chart: Jan-Jul job creation (red intensity gradient)
  - Line chart: entry-level postings with grad supply + peak references

LAYER 03 - FISCAL & ECONOMIC OUTLOOK
  - Hero: City Hall image, 300px, guiding questions
  - "So What" narrative panel
  - 7 metric cards
  - Doughnut chart: tax revenue by source
  - Line chart: 3-scenario revenue projections with toggle selector
  - Grouped bar chart: budget gaps with/without property tax increase
  - Line chart: entry-level jobs vs graduate supply with 2022 baseline
  - Grouped bar chart: cumulative PIT revenue lost

LAYER 04 - MOBILITYSTAT
  - Hero: Meeting image, 300px, guiding questions
  - "So What" narrative panel
  - Metric-level status summary (On Track / Watch / Action / No Data)
  - 7 agency cards with colored avatars, targets, auto-calculated status

DATA & SOURCES
  - Hero: Library image, 260px
  - 12-row source table (cards on mobile)
  - 4 methodology blocks (revenue, jobs, PIT loss, MobilitySTAT)

AI CHAT WIDGET
  - Floating blue button (bottom-right, always visible)
  - 380px panel with suggested questions
  - Sends all dashboard data to Claude claude-sonnet-4-20250514 via Anthropic API
  - Policy analyst system prompt, prose responses
  - Stateless (each question independent)


DESIGN SYSTEM
-------------
Light editorial theme with accent color #1470ad:

  Background:    #ffffff (white)
  Content bg:    #fafafa (off-white for contrast)
  Card bg:       #f9f9f7 (warm off-white, metric cards)
  Card border:   #e8e8e5 / #eeeeeb (subtle warm gray)
  Text primary:  #1a1a1a (near black)
  Text body:     #333333 (dark gray)
  Text muted:    #888888 / #999999 (medium gray)
  Text dim:      #aaaaaa (light gray)
  Accent:        #1470ad (nav active, links, So What borders, chat)
  Blue data:     #2563eb
  Green:         #059669 (positive/on track)
  Red:           #dc2626 (negative/action required)
  Amber:         #d97706 (warning/watch)
  Font:          Inter (Google Fonts), weights 300-800

Hero images with rgba(0,0,0,0.7) overlay. White nav text with
SquareOne logo (CSS brightness invert). Active tab: #1470ad border.
Cards: 12px border-radius, hover shadows with lift. Agency avatars:
colored circles with white initials. Entrance animations: fadeInUp
0.35s with staggered delays.


NAVIGATION
----------
- Hero-integrated nav on every page (scrolls with hero, not sticky)
- White logo + dashboard title (left), 4 tab links (right)
- Active tab: white text, #1470ad bottom border
- Inactive tab: rgba(255,255,255,0.6), hover to 0.9
- Mobile: nav stacks vertically, short tab labels (People, LMI, Fiscal)
- Keyboard shortcuts: 0=home, 1-4=layers
- Smooth fade transition (80ms) between pages
- Footer: SquareOne branding + Data & Sources link


KEY FEATURES
------------
1. Hero image navigation: nav lives inside per-page hero bands with
   background images and dark overlays. No sticky header.

2. Auto-generated "So What" narrative panels: generateFindings() reads
   data and produces conditional sentences. Updates automatically.

3. Pathway tree diagram: HTML/CSS visualization of 72,000 students
   flowing from 9th grade through HS, college, CUNY, to workforce.

4. Education targets on metric cards with auto-calculated status dots.

5. Scenario selector toggle on revenue projection chart (3 scenarios).

6. CSS sparklines in metric cards showing trend direction.

7. Equity disaggregation rows on select metrics.

8. MobilitySTAT agency cards with colored avatars, targets, and
   auto-calculated status (on-track/watch/action/no-data).

9. AI chat widget: floating panel with Anthropic API integration,
   comprehensive data context serialization, policy analyst prompt.

10. Mobile-optimized: responsive hero heights, stacked nav, short
    tab labels, card layouts, sources table-to-cards transform.


CODE ARCHITECTURE
-----------------
1. config.js (gitignored) -- API key via window.ANTHROPIC_KEY
2. isMobile detection
3. Config (LAST_UPDATED, DATA_PERIOD)
4. Color constants (var C = {...})
5. Chart.js global defaults
6. All data as plain JS objects/arrays
7. Helper functions (sentimentColor, trendArrow, buildSparkline,
   cardTargetStatus, metricCardsHTML, sectionHeader, guidingQuestions,
   chartBox, pathBlock, vLine, buildPathwayTree, buildAgencyCards,
   metricStatus, statusDot, imgFallback)
8. Findings engine (findingsPanel, generateFindings)
9. Hero component (buildHero, contentWrap)
10. Page renderers (renderHome, renderPeoplePipeline, renderLaborMarket,
    renderFiscalOutlook, renderMobilitySTAT, renderSources)
11. Headline findings (getHeadlineFinding)
12. Scenario toggle (selectScenario, scenarioColors)
13. Chat widget (initChatWidget, buildDataContext, submitChatQuestion,
    toggleChat, resetChat, chatWelcomeHTML)
14. Navigation (switchPage)
15. Init (keyboard listener, renderHome, initChatWidget)

All HTML built via string concatenation (zero template literals).
Chart instances tracked in activeCharts[] and destroyed on page switch.


DATA SOURCES
------------
All data hardcoded from:
  - NYC Public Schools (graduation results via InfoHub)
  - CUNY (Institutional Research Data Book, ASAP/ACE program data)
  - NY State DOL (Labor Statistics for NYC Region)
  - BLS (Employment Projections 2024-2034, JOLTS state data)
  - NYC Comptroller (budget analysis, JOLTS, PIT reports)
  - NYC OMB (Preliminary and Executive Budget)
  - Center for an Urban Future ("From Degree to Career", March 2026)
  - NYCEDC (State of the NYC Economy, monthly Economic Snapshot)
  - Citizens Budget Commission, Brookings Institution, TNTP
