HOW THE NYC ECONOMIC MOBILITY DASHBOARD WAS BUILT
===================================================

CONTEXT
-------
This dashboard was built for the Office of the Deputy Mayor for Economic
Justice as an internal tool to track education, workforce, and economic
development outcomes for New York City. It is a static data dashboard
(no backend, no authentication) designed to be updated monthly/quarterly
with new data.

The build was done entirely by Claude Code (claude.ai/code) across
multiple sessions, working in a local directory on macOS.

Powered by SquareOne (https://www.squareoneed.com/).


CONSTRAINTS ENCOUNTERED
-----------------------
1. DISK SPACE: The machine had only 138MB free (99% full). A standard
   Vite + React + Recharts + Tailwind project requires ~400MB of
   node_modules. This ruled out npm-based tooling entirely.

2. NODE.JS: The installed Node.js binary was x86_64 on an arm64 Mac,
   so it wouldn't execute. The only working Node binary was bundled
   inside Cursor.app, used solely for syntax validation.

3. RECHARTS CDN FAILURE: The initial approach used React + Recharts +
   Babel loaded from CDN. Recharts' UMD build returned undefined.

4. NESTED TEMPLATE LITERALS: After switching to Chart.js, nested
   backtick strings caused syntax errors. The entire codebase was
   rewritten using pure string concatenation.

5. CHART.JS BAR CHART BUG: Overwriting Chart.defaults.scale.grid with
   a new object wiped internal properties needed for bar rendering.
   Fixed by setting individual properties instead.


SOLUTION: SINGLE-FILE ARCHITECTURE
-----------------------------------
The dashboard is a single index.html file (~730 lines) loading three
CDN resources:

  - Tailwind CSS (cdn.tailwindcss.com) -- utility-first styling
  - Chart.js 4.4.4 (cdn.jsdelivr.net) -- all charts (line, bar, doughnut)
  - Inter font (Google Fonts) -- typography

No React. No JSX. No Babel. No build step. No npm. Just vanilla
JavaScript with string-concatenation-based DOM rendering.

DASHBOARD STRUCTURE
-------------------
The dashboard has a landing page and four layers, navigated via
horizontal tabs in a sticky header bar.

LANDING PAGE (default view)
  - Title block and framing paragraph
  - Four clickable layer cards with dynamic headline findings
  - Each card previews the most important finding from that layer
  - "Powered by SquareOne" footer

LAYER 01 - PEOPLE & PIPELINE
  - Guiding questions + "So What" narrative panel (auto-generated)
  - 10 metric cards with sparklines and equity disaggregation
  - Education pipeline funnel (HTML/CSS, color-coded by health)
  - Education trends line chart with 2019 baseline references

LAYER 02 - LABOR MARKET & SECTORS
  - Guiding questions + "So What" narrative panel
  - 6 metric cards with sparklines
  - Horizontal bar chart: employment change by sector with wages
  - Bar chart: Jan-Jul job creation (red intensity gradient)
  - Line chart: entry-level postings with grad supply + peak references

LAYER 03 - FISCAL & ECONOMIC OUTLOOK
  - Guiding questions + "So What" narrative panel
  - 7 metric cards
  - Doughnut chart: tax revenue by source
  - Line chart: 3-scenario revenue projections with FY2025 baseline
  - Grouped bar chart: budget gaps with/without property tax increase
  - Line chart: entry-level jobs vs graduate supply with 2022 baseline
  - Grouped bar chart: cumulative PIT revenue lost

LAYER 04 - MOBILITYSTAT
  - Guiding questions + "So What" narrative panel
  - Metric-level status summary (On Track / Watch / Action / No Data)
  - 7 agency cards with targets, auto-calculated status dots
  - "Data pending" pulse animation for missing metrics


DESIGN SYSTEM
-------------
Light editorial theme matching SquareOne brand:

  Background:    #fafafa (off-white)
  Header/cards:  #ffffff (white)
  Card border:   #e8e8e5 (subtle warm gray)
  Text primary:  #1a1a1a (near black)
  Text body:     #333333 (dark gray)
  Text muted:    #888888 (medium gray)
  Text dim:      #aaaaaa (light gray)
  Blue accent:   #2563eb (deeper blue for light backgrounds)
  Green:         #059669 (positive trends)
  Red:           #dc2626 (negative trends)
  Amber:         #d97706 (caution)
  Font:          Inter (Google Fonts)

No sidebar. Single-line sticky header with title (left) and
horizontal nav tabs (right). Content centered at max-width 1280px.


NAVIGATION
----------
- Sticky single-line header bar with dashboard title + 4 nav tabs
- Title click returns to landing page
- Tab states: inactive (#888), hover (#333), active (#1a1a1a + bottom border)
- Keyboard shortcuts: 0=home, 1-4=layers
- Smooth fade transition (80ms) between pages
- Footer with SquareOne branding + version/data info


KEY FEATURES
------------
1. Auto-generated "So What" narrative panels: generateFindings() reads
   the data and produces conditional sentences. If data changes, the
   narrative updates automatically.

2. Landing page with getHeadlineFinding(): previews the most important
   finding from each layer on clickable cards.

3. CSS sparklines in metric cards: 3-5 point bar charts showing trend
   direction (green=improving, red=declining).

4. Equity disaggregation: sub-rows on select metric cards showing
   breakdowns (e.g., "Students w/ disabilities: 59% | ELLs: 51%").

5. MobilitySTAT targets: each agency metric has a target value with
   auto-calculated status (on-track/watch/action/no-data).

6. Baseline reference lines on time-series charts (2019 or 2022
   baselines as dashed lines for context).


CODE ARCHITECTURE
-----------------
The JavaScript in index.html follows this structure:

1. Config (LAST_UPDATED, DATA_PERIOD)
2. Color constants (var C = {...})
3. Chart.js global defaults
4. All data as plain JS objects/arrays
5. Helper functions (sentimentColor, trendArrow, buildSparkline,
   metricCardsHTML, sectionHeader, guidingQuestions, chartBox,
   buildFunnelHTML, buildAgencyCards, metricStatus, statusDot)
6. Findings engine (findingsPanel, generateFindings)
7. Page renderers (renderHome, renderPeoplePipeline, renderLaborMarket,
   renderFiscalOutlook, renderMobilitySTAT)
8. Landing page (getHeadlineFinding)
9. Navigation (updateTabs, switchPage)
10. Init (keyboard listener, footer timestamps, renderHome)

Chart instances tracked in activeCharts[] and destroyed on page switch.
All HTML built via string concatenation (zero template literals).


DATA SOURCES
------------
All data hardcoded from:
  - NYC Comptroller (budget analysis, JOLTS, PIT reports)
  - Center for an Urban Future ("From Degree to Career", March 2026)
  - BLS (Employment Projections 2024-2034, JOLTS state data)
  - NYC DOE (Graduation results via InfoHub)
  - CUNY (Institutional Research Data Book, ASAP/ACE program data)
  - NYCEDC (State of the NYC Economy, monthly Economic Snapshot)
  - NY State DOL (Labor Statistics for NYC Region)
